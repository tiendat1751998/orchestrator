# RFC-0033: Confidence Engine

- **Status**: PROPOSED
- **Priority**: P2 — Extended
- **Author**: Orchestrator Architecture Team
- **Created**: 2026-07-03
- **Depends on**: RFC-0010 (Cognitive Layer), RFC-0035 (Capability Graph)

## Summary

This RFC specifies the design of the **Confidence Engine** in AEOS. The Confidence Engine is an active cognitive module inside the `Brain Runtime` that monitors agent confidence scores for specific capabilities (e.g. `crud: 0.99`, `cqrs: 0.42`). If an agent's confidence falls below the configured safety threshold during planning, the Confidence Engine automatically inserts peer review tasks or escalates the task to a second, high-capability agent.

## Motivation

AI models often generate incorrect code when tasked with patterns they are not highly confident in (hallucination).
- Standard orchestrators blindly execute tasks regardless of model limitations, leading to costly failure loops.
- By modeling confidence and implementing automated escalation, AEOS achieves **self-awareness**, preventing failures before execution begins.

## Design

### 1. Architectural Placement

The Confidence Engine intercepts the candidate plans generated by the Planner and modifies the DAG structure prior to final scheduling.

```
  Planner Candidate ──► [Confidence Engine] ──(Low Confidence?)──► Insert Reviewer ──► Schedule
```

---

### 2. Contracts (`contracts/brain/confidence.go`)

```go
package brain

import (
	"context"
	"github.com/tiendat1751998/orchestrator/contracts/fsm"
)

// AgentConfidence represents the capability rating of an agent.
type AgentConfidence struct {
	AgentID     string  `json:"agent_id"`
	Capability  string  `json:"capability"` // "crud", "review", "refactor"
	Score       float64 `json:"score"`      // [0.0, 1.0]
}

// ConfidenceEngine evaluates plan safety thresholds.
type ConfidenceEngine interface {
	// Evaluate assesses a candidate plan DAG and modifies it if confidence criteria are unmet.
	Evaluate(ctx context.Context, dag fsm.DAG, threshold float64) (fsm.DAG, error)
	
	// GetConfidence retrieves the confidence metrics of an agent.
	GetConfidence(ctx context.Context, agentID, capability string) (*AgentConfidence, error)
}
```

## Impact

- **Self-Correcting DAGs**: If the assigned coder agent has a low score for a complex pattern (e.g., CQRS: 0.42), the Confidence Engine automatically mutates the execution graph to append a `ReviewCode` task assigned to a high-capability reviewer agent.
- **Accurate Resource Routing**: The Planner uses confidence scores to select the most cost-effective agent that still meets the minimum safety threshold.
